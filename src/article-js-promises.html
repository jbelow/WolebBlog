Introduction To JS Promises

by Joshua Below

Posted on September 20, 2020 at 3:40 PM

A promise is a way to process asynchronous operations. An example of when you would want to use this is if you are grabbing different pieces of information from a database and you want them to all load at the same time.

A simple way to think about promises is in the world when you ask a friend to promise that they will help you get something. First, you have to ask your friend if they are willing to help you (this is calling the promise or pending) then you have to wait for their response. Your friend will most likely get back to you and say that they are able or unable to help (this would be fulfilled or rejected).

You might be requesting information from the database and well it's pending the database is checking if you can fulfill the tasks that you requested. If it cannot find the correct data it will reject the request, but if it can successfully find it it will be fulfilled.


To create a simple example for a promise, we can use an if else statement to represent if a file was successful uploading or not. 

//using a function is not needed but a way to show how you can use one to call a promises
function MathPromise() {
    let numberOne = 1;
    let numberTwo = 5;

    //you can either set a promise to a variable or just return it like we are here
    //here we are either passing resolve or reject when the function is being called
    return new Promise((resolve, reject) => {
        if (numberOne + numberTwo === 15){
            resolve ({
                message: "10 and 5 added = 15!"
            })
        } else {
            reject({
                message: "the numbers didn't equal 15 :("
            })
        }
    })
}

//this is where we check to see if it passed or not with .then or .catch
//.then means now that we have a successful request what do should the code do now
//.catch means that there was an erorr and it was rejected and the code needs to know what to do now
MathPromise().then(message => {
    console.log(message);
}).catch( failed => {
    console.log(failed.message);

})

You can also have multiple promises and have them processed all together in a group (If you're familiar with callbacks, then using promises fixes the problem of callback hell).

This is the simple syntax for creating promises. There are of course more options for now put:

Promise.all()

Gives an array of all the inputs from the promises if they all are resolved


Promise.allSettled()

Gives an array of all the inputs from the promises regardless if they are resolved or rejected.


Promise.race()

As it suggests, the race() is used for if you want to have the return as soon as the first promise has completed.


There are a few more, but from my observations these are the main ones to use.

conclusion

There is obviously more complexity comparatively to an if else statement, but from a more simple standpoint, this is how they work.
